# Amazon RDS: Managed Relational Databases

**Amazon Relational Database Service (Amazon RDS)** makes it [easy to set up, operate, and scale](#yellow) a [relational database](#green) in the cloud.

### What is it?

RDS manages the [tedious tasks](#yellow) of database administration ([backups](#purple), [software patching](#purple), [automatic failure detection](#purple), and [recovery](#purple)) so you can focus on your [application](#green).

### Key Features

- [Multi-Engine Support](#green): Supports [PostgreSQL](#purple), [MySQL](#purple), [MariaDB](#purple), [Oracle Database](#purple), [SQL Server](#purple), and [Amazon Aurora](#purple).
- [High Availability](#green): [Multi-AZ deployments](#underline) allow for [automatic failover](#yellow) in case of [hardware failure](#purple).
- [Performance](#green): [Provisioned IOPS](#purple) storage for [fast, predictable performance](#yellow).
- [Security](#green): [Network isolation](#yellow) using [VPC](#purple), [encryption at rest](#underline), and [encryption in transit](#underline).

### Use Cases

- Traditional web applications (e-commerce, CMS).
- Enterprise applications (ERP, CRM).
- Mobile and online games requiring a relational backend.

### Why use it?

You get the power of a [familiar relational database](#green) without the [headache](#yellow) of managing the [underlying infrastructure and maintenance](#underline).
